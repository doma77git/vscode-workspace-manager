============================================================
VS Code Workspace Manager — One-Time Deploy Commands
============================================================

Prerequisites (check):
  pwsh --version          (PowerShell 7+ required)
  git --version
  code --version          (VS Code CLI must be in PATH)

One-time setup (run from any directory):
------------------------------------------------------------

1) Initialize the git repository and make first commit:

   pwsh -NoProfile -ExecutionPolicy Bypass -File ''C:\VSCode\Templates\scripts\Init-TemplatesRepo.ps1''

   This will:
   - Create .gitignore, README.md, sample template, sample profile
   - git init, git add ., git commit ''Initial commit: templates repo''
   - Install the pre-commit hook

2) Launch the interactive Workspace Manager:

   pwsh -NoProfile -ExecutionPolicy Bypass -File ''C:\VSCode\Templates\scripts\WorkspaceManager.ps1''

3) (Optional) Run the GitHub Actions workflow locally:

   act -W .github/workflows/validate.yml

============================================================
Post-deploy verification checklist:
============================================================

[ ] .gitignore exists and excludes secrets
[ ] .git/hooks/pre-commit blocks commits with passwords/tokens
[ ] .github/workflows/validate.yml passes JSON lint + secrets scan
[ ] WorkspaceManager.ps1 menu loads and all options respond
[ ] code --profile <name> <workspace> opens correctly
[ ] meta/deepseek-byok.json placeholder present (no real keys)
[ ] meta/trust.json has emptyWorkspaceTrust = false
